Tic Tac Toe

by: Preston Seay

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make this game because I thought that tic tac toe was a classic game and I wanted to get a feel for the code behind it.

2. What I Learned

I learned that color gave the user a more positive experience and that if I were to add more color and change the colors, people would have more fun.

3. Architecture of My Game

I have the main class that runs the window and sees if something is clicked, then I send that input to another class that will decide what to do with that click(nothing/place x or o). I also have a class that would do all the checking and calculating.

4. Technical Implementation

I was proud that I got through the project without using 2 dimensional arrays or enums. But I had to use switches. I used an array of buttons to store the button information. I made a string switch between "x" and "o" to alternate between the two players' turns.

5. Most Challenging Part

The only challenge of the project was deciding between two ways of making a game. The way that involves reading where the mouse clicks and assigning it to a "button" which would be an 3rd of the height and width(roughly). I used the way where I created an array of buttons and had an actionlistener for them.

6. Future Fixes and Updates

I could make an ultimate tic tac toe.
